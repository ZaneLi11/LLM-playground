In 2026, I started a tiny project called “Signal & Noise” (S&N). The goal: turn messy notes → clean ideas. I wrote: “Less scrolling, more thinking!” and pinned it above my desk. Then I asked myself: what counts as useful info vs. random chatter?

My rule was simple: if a line answers who/what/when/why/how, it stays; if it’s just vibes… it goes. Example: “Meeting @ 3:30pm (Room 217)” ✅, but “lol that was wild” ❌. I also tag items like this: [TODO], {RISK}, (NOTE), <LINK>, and #topic.

Sometimes I compress thoughts into one-liners: plan -> execute -> review -> repeat. Other times I dump raw fragments: “battery 20%… Wi-Fi dead… printer jam… ugh.” The trick is to keep separators consistent: commas, semicolons; colons: dashes— and even pipes | when needed. Consistency beats perfection.

When I read text, I look for patterns: numbers (1, 2, 3), ranges (10–20), and ratios (3:1). I also watch for units: kg, lb, °C, $, %. A sentence like “Revenue +12.5% (QoQ)” tells a story faster than three plain sentences. Symbols can be shortcuts—if you don’t overdo them.

I test my loader with weird strings: quotes " " and ' ', smart quotes “ ”, brackets [ ], braces { }, and parentheses ( ). I add slashes A/B, backslashes C:\path\file, and URLs like https://example.com?a=1&b=2. If parsing breaks here, it will definitely break in real data.

Finally, I keep a “checksum paragraph” full of edge cases: ~!@#$%^&*()_+ and []{}<>. I include math signs too: x ≥ y, a ≠ b, Δt = 0.5s, π ≈ 3.14159. If the text survives this stress test, I trust the pipeline.
